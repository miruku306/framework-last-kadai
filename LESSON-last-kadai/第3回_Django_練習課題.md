## 1. 開発ミッション
あなたは会社のシステム担当者です。前半の「在庫管理」の仕組みを応用して、社内のメンバー一覧を確認できる**「デジタル社員名簿」**を新規に開発してください。

## 2. 準備ルール（重要）
*   **新しく別のアプリ**を作って作業してください。第3回 Django 授業資料で使用したフォルダを複製してそのフォルダで作業します。（授業と実習はコンテナを分けてください。　**コンテナの名称も変えること**）
*   エラーが出た場合は、ブラウザのエラー画面をよく読み、前半の「在庫管理」のコードと見比べてください。

## 3. 要件定義（仕様書）

### ① アプリ名・URL
*   **アプリ名**: `staff`
*   **全体の入り口URL**: `/portal/`
*   **名簿一覧のURL**: `/portal/list/`
*   **個人詳細のURL**: `/portal/user/<int:user_id>/`

### ② データ仕様
`staff/views.py` の中に、以下のデータ（辞書のリスト）を定義して使用してください。
```python
STAFF_MEMBERS = [
    {'id': 1, 'name': '山田 太郎', 'dept': '営業部', 'role': 'マネージャー', 'email': 'yamada@example.com'},
    {'id': 2, 'name': '佐藤 華子', 'dept': '開発部', 'role': 'エンジニア', 'email': 'sato@example.com'},
    {'id': 3, 'name': '伊藤 健太', 'dept': '開発部', 'role': '一般', 'email': 'ito@example.com'},
    {'id': 4, 'name': '鈴木 一郎', 'dept': '人事部', 'role': '一般', 'email': 'suzuki@example.com'},
]
```

### ③ 機能要件（必須）
1.  **共通デザインの適用**: 前半で作った `base.html` を継承（extends）し、ナビゲーションが表示されていること。
2.  **一覧表示機能**: `/portal/list/` にアクセスすると、全社員の「氏名」「部署」がテーブル形式で表示されること。
3.  **詳細表示機能**: 一覧の氏名をクリックすると、その社員の全情報（部署・役職・メールアドレス）が表示される詳細ページへ飛ぶこと。
4.  **条件付きデザイン**: 役職（role）が「マネージャー」の社員は、名簿一覧で名前の横に記号やマーク「👑」などを表示するか、名前を**太字**にすること。

---

## 4. 制作ステップ（推奨手順）

1.  **アプリを作る**: `python manage.py startapp staff`
2.  **設定に登録**: `myproject/settings.py` の `INSTALLED_APPS` に `staff` を追加。
3.  **URLを繋ぐ**:
    *   `myproject/urls.py` に `path('portal/', include('staff.urls'))` を書く。
    *   `staff/urls.py` を新規作成して、一覧と詳細のパスを書く。
4.  **Viewを書く**: `staff/views.py` に `STAFF_MEMBERS` を定義し、2つの関数（一覧用・詳細用）を作る。
5.  **テンプレートを作る**:
    *   `templates/staff/list.html` を作成。
    *   `templates/staff/detail.html` を作成。

---

## 5. ヒント：詳細ページの「探し方」
特定のIDの社員を探すには、以下のコードを参考にしてください。
```python
# views.py の詳細用関数の中
user = next((s for s in STAFF_MEMBERS if s['id'] == user_id), None)
```
